## 它解决什么问题

事情是这样的。你读过毛选，觉得里面的方法论说得很对——矛盾论、实践论、持久战。但真遇到一个商业决策、一个团队管理问题、一个竞争困局的时候，这些方法论怎么"落地"？你不可能翻回原著查章节。

普通做法是凭印象说一句"不就是实事求是嘛"，但这等于没说。这个 Skill 做的事不一样：它把《毛选》1-4卷（122+篇、140万字）蒸馏成 **6个心智模型 + 12条决策启发式 + 8种经典句式**，并且打包成一套**可执行的思维操作系统**。

你问一个问题，它先诊断矛盾在哪，再用对应的模型给方案。全程用毛选的语言风格，但不是背语录——是真的在分析。

## 它能交付什么

| 你遇到的问题 | 它会调用的模型 | 交付物 |
|-------------|:---:|-------|
| 局面混乱，不知道该抓哪个 | 模型一：矛盾分析法 | 列出主要矛盾 → 判断矛盾主要方面 → 行动优先级 |
| 需要做决策但信息不全 | 模型二：实践-认识循环 | 要求补调查 → 先感性后理性 → 给可验证的初始方案 |
| 长期竞争，看不到头 | 模型三：持久战三段论 | 判断当前阶段（防御/相持/反攻）→ 阶段策略 |
| 资源有限，想做太多 | 模型四：集中优势兵力 | 做减法 → 选突破点 → 集中力量单点打透 |
| 多方博弈，立场模糊 | 模型五：统一战线策略 | 画利益地图 → 分敌/友/中间派 → 给撬动策略 |
| 团队意见分散，统一不了 | 模型六：群众路线 | 收集分散意见 → 系统化 → 试点验证 → 迭代 |

```bash

```


- 毛选思维怎么看…
- 帮我用持久战的思路…
- 统一战线模型分析…
- 集中优势兵力，这个怎么选…

**场景触发（说场景，自动路由到对应模型）**：

- "我有个商业竞争问题" → 自动路由到模型一+五
- "团队内部矛盾怎么处理" → 自动路由到模型五+六
- "这个项目做了两年，感觉拖不动了" → 自动路由到模型三
- "新产品方向太多，不知道怎么选" → 自动路由到模型四

## 场景路由表

Skill 内部根据问题特征自动选模型。理解这个路由逻辑后，你也可以直接点名用哪个模型：

| 你的问题特征 | 推荐模型 | 一句话操作 |
|-----------|:---:|---------|
| 局面复杂，看不清主次 | 模型一：矛盾分析法 | "主要矛盾是什么？" |
| 信息不足，怕做错决定 | 模型二：实践-认识循环 | "先调查，再做结论" |
| 长期内耗，节奏紊乱 | 模型三：持久战三段论 | "现在处于哪个阶段？" |
| 资源不够，四面楚歌 | 模型四：集中优势兵力 | "断其一指，不要伤其十指" |
| 阵营不清，敌友不分 | 模型五：统一战线策略 | "谁是敌人？谁是朋友？" |
| 决策需要群众基础 | 模型六：群众路线 | "从群众中来，到群众中去" |

## 安全边界

### 不会做什么
- **不输出政治立场判断**。Skill 聚焦方法论层面（矛盾分析、阶段判断、资源聚焦、利益博弈），不涉及具体历史事件评价或当代政治观点输出。
- **不替代专业咨询**。法律、医疗、投资等需专业资质的决策，本 Skill 不提供建议。
- **不泄露用户隐私**。所有分析只基于用户主动输入的问题，不收集、不存储、不转发。

### 什么时候会停下来问你
- 你的问题超出了方法论适用范围（如纯技术问题、纯数学计算）
- 你的表述过于模糊，无法判断主要矛盾
- 当你要求输出可能违反平台政策的内容时

### 不该用这个 Skill 的情况
- 你需要快速事实查询（查日期、查数据）→ 用搜索
- 你需要代码实现或技术支持 → 用对应的技术 Skill
- 你需要情绪支持或心理咨询 → 不适用
- 你只想看看毛选语录 → 直接搜原文更快

## 验证与测试

装完用这句验收：

```
用毛选的视角分析：一个创业公司面临资源有限、竞争激烈的局面，应该先做什么后做什么？
```

**合格表现**：
1. 先诊断"主要矛盾是什么"（不是直接给方案）
2. 引用模型四（集中优势兵力）或模型一（矛盾分析）给具体步骤
3. 语言是白话文风，有设问自答或排比递进
4. 有一句话结论，掷地有声
