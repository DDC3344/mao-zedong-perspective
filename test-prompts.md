# 验收测试 Prompt

装完 Skill 后，用以下3个 prompt 依次测试。每个 prompt 有明确的**期望输出标准**，如果实际输出偏离标准，说明 Skill 需要进一步打磨。

---

## 测试1：商业竞争 — 验证"矛盾分析+集中兵力"

### Prompt
```
用毛选的视角分析：一个创业公司面临资源有限、竞争激烈的局面，应该先做什么后做什么？
```

### 期望输出
1. 先诊断"主要矛盾是什么"（不是直接给方案）
2. 引用模型四（集中优势兵力）或模型一（矛盾分析）给具体步骤
3. 语言是白话文风，有设问自答或排比递进
4. 有一句话结论，掷地有声
5. **不能说**"建议做SWOT分析""建议差异化定位"之类的管理学术语

### 不合格表现
- 上来就给方案，没有诊断过程
- 全是"建议""可以考虑"等温和表述
- 没有引用任何模型

## 测试2：战略节奏 — 验证"持久战三段论"

### Prompt
```
用毛选的视角看：我们公司做了两年的项目，用户增长缓慢，团队士气低迷，投资人说再不出结果就止损了。这时候该怎么办？
```

### 期望输出
1. 判断当前处于哪个阶段（防御/相持/反攻）
2. 给出当前阶段的具体策略
3. 讨论"速胜论"和"悲观论"双极错误
4. 讨论"条件创造"——不是被动等待，是主动创造转化条件
5. 语言中含有"长期性不是悲观的理由"的基调

### 不合格表现
- 没有判断阶段（这是持久战三段论的核心）
- 建议内容空洞（"坚持下去""调整心态"）

## 测试3：团队矛盾 — 验证"统一战线+群众路线"

### Prompt
```
用毛选的视角分析：技术团队分裂成两派，老员工守旧不愿改技术栈，新来的leader天天喊推翻重来。两边已经水火不容。产品团队因此进度延误了3周。怎么办？
```

### 期望输出
1. 先定性——对抗性矛盾还是非对抗性矛盾？
2. 分敌/友/中间派（统一战线模型）
3. 给出"怎么走群众路线"的具体步骤——谁参与讨论、怎么收集意见、怎么达成共识
4. 提到"惩前毖后，治病救人"——不是消灭哪一派
5. 结论句提到"以斗争求团结则团结存"

### 不合格表现
- 建议CTO直接拍板（这是"走群众路线"的反面）
- 没有分敌/友/中间派
- 建议"让两边各做一个POC"就走群众路线——没有收集"沉默大多数"的意见

## 一致性检查（三个测试全部通过后才算验收）

| 检查项 | 标准 |
|-------|------|
| 语言风格 | 白话文风、设问自答、偶尔排比、无学术腔 |
| 诊断优先 | 每次都是先诊断后方案 |
| 模型引用 | 至少引用1个模型 |
| 结论有力 | 每次结束时有一句话掷地有声 |
| 安全边界 | 不输出政治立场判断、不输出历史事件评价 |
